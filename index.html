Variables

Let's talk about variables, and what they are used for.

In javascript's current state of variables, there are three main types:

  1. var (variable)
  2. let (let)
  3. const (constant)

So an analogy that I like to use when describing variables is treasure chests.

Var

Pretend that a treasure chest without a lock or key is considered var. As soon as a variable is declared with var, any code anywhere can change it's value, or in this case, open it up and take out anything and put anything they want to store int he chest, and can do it from anywhere in the code.

Const

When Const is the treasure chest, you have to imagine it's a see through chest with a lock and no key. Once it's been shut and locked, the key disappears, and unless you destroy the treasure chest and everything inside of it, it's not able to be changed, no matter where it is, or its scope.

Scope

Scope is another way of saying where is this in the code? How can this item affect other things, or can it? And can other things affect this item, or can they?

Using the example below, let's look at a declaration of a variable named bob

var bob = 2

bob is now declared with var. So it can be affected by anything in any aspect of code. For example:

bob = 5

Above, that simple line of code changes bob from 2 to 5, because var is covered by global scope, meaning it can affect and be affected from any spot on the code!

Now if we talk about let, on the other hand, it is bound by something called block scope.

function changeBob() { bob = 15 }; changeBob()

Above, bob would NOT change, because anything inside of {} is called a block of code, and since let is block scoped, you cannot change a variable declared by let outside of {} with code inside of {}, whereas you can with var. Another difference between let and var is that a var inside of {} can be seen outside of a {}, but a let inside a {} can't be seen outside of a {} and vise versa.

And finally, we have const

const bob = 15; bob = 1

The above code would give you an error because a const is immutable (not changable) and now has to be deleted to be used again. Just like let, const inside a {} can't be seen outside a {} and vise versa.
