Variables

Let's talk about variables, and what they are used for.

In javascript's current state of variables, there are three main types:

  1. var (variable)
  2. let (let)
  3. const (constant)

So an analogy that I like to use when describing variables is treasure chests.

Var

Pretend that a treasure chest without a lock or key is considered var. As soon as a variable is declared with var, any code anywhere can change it's value, or in this case, open it up and take out anything and put anything they want to store int he chest, and can do it from anywhere in the code.

Let

The difference between var and let is scope. I'll explain scope in the next section, but consider a treasure chest with let is like a chest laid on a land mass that can be viewed and defined by anyone anywhere else, but since it's not on the same land you're standing on, you can't actually take anything out or put anything into it. But if it's on the same landmass as you, you can also

Const

When Const is the treasure chest, you have to imagine it's a see through chest with a lock and no key. Once it's been shut and locked, the key disappears, and unless you destroy the treasure chest and everything inside of it, it's not able to be changed, no matter where it is, or its scope.

Scope

Scope is another way of saying where is this in the code? How can this item affect other things, or can it? And can other things affect this item, or can they?

Using the example below, let's look at a declaration of a variable named bob

var bob = 2

bob is now declared with var. So it can be affected by anything in any aspect of code. For example:

bob = 5

Above, that simple line of code changes bob from 2 to 5, because var is covered by global scope, meaning it can affect and be affected from any spot on the code!

Now if we talk about let, on the other hand, it is bound by something called block scope.

function changeBob() { bob = 15 }; changeBob()

Above, bob would NOT change, because anything inside of {} is called a block of code, and since let is block scoped, you cannot change a variable declared by let outside of {} with code inside of {}, whereas you can with var. Another difference between let and var is that a var inside of {} can be seen outside of a {}, but a let inside a {} can't be seen outside of a {} and vise versa.

And finally, we have const

const bob = 15; bob = 1

The above code would give you an error because a const is immutable (not changable) and now has to be deleted to be used again. Just like let, const inside a {} can't be seen outside a {} and vise versa.

Just as a final reminder!

Block scope!

{const bob = 2}

Global scope!

const bob = 2

Arrays

Important note about Arrays:

Soon we'll talk about objects. For purposes of when you're trying to learn later coding methods, keep in mind that arrays ARE objects, but they are a specific object that is designed to what they do, and not like a standard object

Data Storage

Amongst many things, arrays are ways to store data. Like variables are ways to store instances of data, arrays are objects that store multiple forms of data, including declared variables. Here's an example:

const bob = 12; const jimmy = 14

Now, if you were to take those to and make an array to hold boty bob and jimmy called, let's say, car, for example...:

const car = [bob, jimmy]

So as you can see, declaring an array is similar to declaring a variable, but when you put [] there, it tells javascript you're declaring an array. Arrays can be left empty as well, but for the case of this basic explanation, let's just use our car.

You might have noticed I declared an array with const and not var, or let. There is a reason for this. You remember how we discussed that const cannot be changed? Well since we used const we cannot change WHAT IT IS that we declared, but since an array is an object, we can still modify what's INSIDE of the array. But if we would have used let or var, which let you change the data they are declared with, you can declare it like this:

let car = [bob, jimmy]; car = 4

The above could cause some serious problems! Since let allows us to change the type of thing we're declaring, by using a lie of code 'car = 4', now you've changed what car is. It's not an integer, 4, instead of an array. If you have code later on relying on what's inside of car, it will break your task because now car is 4 instead of the value of bob and jimmy, 12, and 14.

Index

An important part of accessing the data you've stored in an array is index. So let's say we want to make a NEW variable, and make it equal to bob from our car array. How would we do that? Well, remember that car is [bob, jimmy], which means that the first item is bob. When we are trying to grab an item from an array, we have to use the same brackets we did when we created our array, []. So once we use those, then we have to tell it which number item (it's index) we're getting. BUT THAT'S NOT ALL! Arrays work on indexed counting, which starts with 0, not 1. So in order to get bob's value out of car, we do this:

const bobbyAge = car[0]

By doing the above, we now have bobbyAge set to 12, because car[0] is grabbing the first item out of car, which is 12! We could do the same for jimmy by car[1] instead of 0!

Objects

Data Storage

Much like Arrays, Objects also store data. They can store data in a different way, but also more organized than an array as well, in the event you need such. Let me show you an example:

              const car = {
                  make: 'ford',
                  model: 'mustang',
                  color: 'red',
                  year: 2018,
                  doors: 2,
              }

Much like arrays require [], objects require {}

Key/Value

Objects, as you can see, don't just say {make, model, color, year, doors} like an array. This both makes it more understandable, but also showcases a way that instead of declaring a bunch of variables and putting their names inside an array, like with bob and jimmy in the array example in the previous section, you can both declare and define what bob and jimmy are inside an object at once. So instead of having:

              const bob = 12;
              const jimmy = 14;
              const car = [bob, jimmy]

Instead, we can do this:

              const car = {
                  bob: 12,
                  jimmy: 14
              }

This way we don't need to declare, then also store it in the array, and we can just have it all inside the object.

Inside of the car object above, bob and jimmy are considered keys, and 12 and 14 are considered values. You declare the key, and then after a : describe its value. You can do much more with object and arrays both than what is shown here, but I encourage you to explore that on your own.

Accessing Object data

You can do a similar method to what you did with arrays, using the [], but by default, objects don't recongize indexed numbers. Instead, you use what's called index notation. The other way is called dot notation. I'll show you examples of each.

Index notation

car["bob"]

Dot notation

car.bob

Loops

Loops are an important method in javascript. There are a few, but for these basic lessons, I'll start with 2. The while loop, and the for loop. Both have a time when they are useful over the other, but experimenting with them is always good.

For loop

A for loop is something you can kind of use in english language. For our example, lets start with a variable and a number

const maxNumber = 10

So now we have a max number declared, and can use that as an anchor to tell our loops when to stop.

With that in mind, let's show you the basic code syntax for a for loop so you can get an idea of what's happening.

              for (let applesIHave = 0; applesIHave < maxNumber; applesIHave++) {
                console.log(applesIHave)
              }

Let's break this down.

for is a keyword that lets javascript know you're going to use a for loop. I wrote applesIHave to help make this more english'y. Normally, you would say index, or i, instead there, so it's more concise. However, it's not necessary to name it something other than i if you know what you're doing. But for the sake of learning, let's use it so we know it's representing our apples. So the breakdown goes like this, from left to right, on the top line of the for loop block.

for**for keyword to explain it's a loop** (let applesIHave = 0 **basically saying for the amount of apples I have....** ; apples I have < maxNumber; **while the amount of apples I have are less than maxNumber(which is 10 that we declared above)... ** applesIHave++) ** make applesIHave go up by one. In Javascript ++ is a mathematical expression that takes whatever number you have and increases it by one, for reference.

There's a great tool you can do to help you map this out visibly in case that's how you need to learn as well. Thinking about the for loop, consider all the parts just described. Also, so you know, the console is in your browser, and any time console.log() happens, it will print results to your console. A wonderful learning/testing/debugging tool you should make liberal use of.

When the condition of your loop is met, and a condition is met as 'false', it will exit completely and stop. So it's important to remember the way you write things is important. Let me show you an example of where this can go wrong.

              for (let applesIHave = 9; applesIHave < maxNumber; i--) {
                  console.log(applesIHave)
              }

In the for loop above, remember syntax and what it means. I have 9 apples. is 9 < maxNumber?(which is 10) so yes. Then subtract 1 from 9 (-- is just like ++, but for subtract instead of add). Can you see a possible problem here? Just in case you don't just yet, that's okay! But remember, the condition that the loop is waiting to change to false before the loop can stop is that applesIHave < 10. But since you started applesIHave at 9 and told it to subtract by 1 each time the loop happens. This will ensure that the number will never be greater than 10, and the function will never stop. This is traditionally called an endless loop. This will freeze up your browser/app/etc., and you will have to close your browser and reopen it again.

While loops

While loops are a lot like for loops, but written much shorter:

              const bob = 2;
              while (bob < 10) {
                  console.log(bob)
              }

As you can see, this would do much the same thing as the for loop example above, but written differently. These are just as susceptible as the for loop to become an endless loop, though, and freeze all your browsers/apps/etc..

Functions

While variables are important as they give us the building blocks of what to operate with when trying to build something, functions would be akin to the tools that we use to put stuff together. Functions give us the ability to use many different parts to give an end result that we often see as a user on a web page or app.

There are a few different ways you can write functions in javascript, and that is in part to different versions being made and released over the years. However, I'll show you the way that I find easiest and fastest.

              const multiplyByTwo = () => {
                  console.log(4 * 2)
              }

In programming languages, your functions are set and ready, but not firing off until you 'call' them. So in the above example, the function multiplyByTwo is taking a simple function in form of mathematical operation of 4 * 2. As we know, that will result in 8, but to make it print to the console, you will have to call it like this:

multiplyByTwo();

() like that after a function name is what initializes a function and runs the code we've written inside.
